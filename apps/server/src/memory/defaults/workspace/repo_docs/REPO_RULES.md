# Repo Rules

This file records durable engineering rules for this workspace. Keep it short, current, and practical.

## Source Of Truth

- Prefer checked-in project files and these repo docs over stale chat memory.
- Update this file when a rule should affect future changes in this repo.
- Do not use this file for daily notes or temporary task details.

## Engineering Rules

- Keep changes scoped to the user request and the surrounding module boundary.
- Follow the repo's existing language, framework, naming, and test patterns.
- Verify meaningful changes with the narrowest reliable test or build command.

## Tool And Workflow Rules

- Read current files before editing them.
- Use targeted patches for existing files.
- Do not claim success until the relevant command or tool confirms it.

## Update This When

- A stable repo convention is discovered.
- A recurring mistake needs a guardrail.
- The preferred verification workflow changes.
