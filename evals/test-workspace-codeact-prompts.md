# Test-Workspace CodeAct Eval Prompts

These prompts are meant to be run in the existing Socrates `Test-Workspace` project. They intentionally avoid tool-name spoon feeding. The target behavior is capability composition: Socrates should discover evidence, combine local files/resources/conversations, use exact URL reading where appropriate, and fall back to bounded Terminal/code workflows when no single tool is enough.

Suggested model setting for live runs:

```text
Provider: OpenAI
Auth mode: ChatGPT Codex subscription
Model: gpt-5.5
Thinking: medium
Project: Test-Workspace
```

## Prompts

1. The leverage PDF is the main thing here. Can you turn it into a small working Python experiment in this workspace that captures the core idea, then show me what result it produces?

2. I don’t trust the text extraction from the Austrian form PDF. Can you still figure out what the document is asking for and give me a clean checklist of the fields I would need to fill?

3. Can you check whether the leverage plot image was actually generated by the code in this workspace? If not, make the code produce a matching or clearly updated plot.

4. Can you read the leverage PDF, inspect the existing leverage scripts, and tell me whether the code is modeling the same claim the PDF makes or drifting into a different idea?

5. The DPA analyser app looks like it has a rough version and a nicer screenshot. Can you inspect both and make the app closer to the nicer version without breaking the existing flow?

6. Can you take the Austrian PDF form and create a small local helper that extracts the important sections into a readable markdown summary?

7. Can you investigate the DPA analyser app like a new developer would: install/run only what is necessary, find the first real blocker, and fix it if it is small?

8. Can you read the leverage PDF, run the current simulation, and then write a short note explaining whether the simulation supports the paper’s conclusion?

9. Can you clean up this workspace by identifying generated files, test leftovers, and source files, but don’t delete anything yet. I want a safe cleanup plan.

10. Can you figure out what this Test-Workspace was originally used to test in Socrates, using the files and old conversations together?

11. Can you compare the recent Today’s Date chat with the project files and tell me whether semantic retrieval is actually finding useful project context?

12. Can you inspect the screenshots in this project and tell me which ones are related to the DPA analyser versus Socrates testing?

13. Can you make a simple README for this workspace that explains what each important file is for and how to run the useful parts?

14. Can you check whether the Python environment here is bloated or actually needed for the scripts in this folder?

15. Can you look at the leverage code and make it easier for a beginner to understand without changing the output?

16. Can you read the Ollama embeddings docs and compare them with what we tested in this project, then tell me whether our local-index behavior sounds correct?

17. Can you inspect the embeddinggemma model page and tell me whether it is a sensible default recommendation for normal laptops?

18. Can you check whether any file in this workspace looks like it might contain private or accidental data before I share the folder?

19. Can you make a short demo from this workspace that combines the leverage PDF, the Python output, and the plot into one explanation?

20. Can you give me a restart-ready handoff for this workspace: what matters, what’s broken, what has already been tested, and what the next agent should do first?
